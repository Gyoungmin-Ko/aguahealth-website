# AI 솔루션 가격·수익 구조 설계 가이드

## 1. 수익 구조를 만들기 위해 알아야 할 것

### 1.1 고객 세그먼트와 가치 인식
- **국내 중소 제약·바이오 CEO**: 예산 제약이 크고, “글로벌 수준 전략”은 원하지만 대형 컨설팅 비용은 부담. **낮은 진입 장벽 + 빠른 체감 가치**가 중요.
- **국내 지사(글로벌 본사 산하)**: 본사 예산·프로세스에 맞춰 **프로젝트 단위 견적·결재**를 선호. 원타임·패키지가 구독보다 수용되기 쉬움.
- **시장진입/경제성평가 담당자**: 반복 작업(리포트, 표, 슬라이드 초안)을 **AI로 줄여 시간 절약**하는 것에 가치를 둠. “건당 비용” 또는 “월 N건 포함”이 이해하기 쉬움.

### 1.2 수익 발생 구조 유형
| 유형 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **구독(Subscription)** | 월/연 고정료, 사용량 무관 또는 캡 있음 | 예측 가능한 매출, LTV 확보 | 초기 전환 장벽, 이탈 시 매출 급감 |
| **원타임(Project/One-time)** | 프로젝트·건당·모듈당 요금 | 결재·예산 통과 쉬움, 첫 구매 진입 용이 | 매출 변동성, 업셀 전까지 단발 |
| **하이브리드** | 구독(기본 사용권) + 초과분/추가 프로젝트 과금 | 진입은 원타임, 안정은 구독 | 가격 설계·소통 복잡도 증가 |
| **성과 연동(Value-based)** | 등재 성공·절감액의 % 등 | 고객과 리스크 공유, 프리미엄 근거 | 측정·계약 복잡, 현금흐름 불안정 |

**권장**: 국내 CEO·지사 전환을 위해 **하이브리드(구독 + 원타임)**를 기본으로, “먼저 한 건 써보기”를 가능하게 하는 **원타임 단품**을 전면에 내세우기.

### 1.3 가격 설계 시 필수 확인 사항
- **경쟁 대안의 가격대**: 글로벌 컨설팅(IQVIA, Parexel 등)은 맞춤 견적 위주로 공개 가격이 거의 없음. 국내 대형 컨설팅·컨사장 도입 비용(프로젝트당 수천만원~억 단위)을 **앵커**로 활용해 “그 일부 수준”을 강조.
- **고객의 예산 사이클**: 연간 예산 확정 시점, 프로젝트 단위 결재 한도.
- **가치 지표**: “시간 절감 N시간”, “리포트 1건 당 비용”, “등재 지원 1건 당 비용” 등 **구체적 결과**로 가격을 설명할 수 있어야 함.
- **전환 경로**: 무료 체험/데모 → 원타임 1건 → 구독 또는 추가 건수.

## 2. 글로벌·국내 참고 사례

### 2.1 글로벌 (Pricing & Market Access 컨설팅)
- **IQVIA, Parexel**: 프로젝트·전담팀 기반 **맞춤 견적**. 공개 단가는 없으며, 가치 스토리·HEOR·HTA·페이어 접근을 패키지로 제공. 국내 중소기업이 직접 쓰기엔 비용 부담이 큼.
- **시사점**: “글로벌 수준”을 **모듈·건별로 쪼개서** 제공하면, “한 건만 써도 된다”는 메시지로 진입 장벽을 낮출 수 있음.

### 2.2 헬스케어·제약 SaaS
- **Per-seat**: Salesforce Health Cloud 등 사용자당 월 과금. 팀 단위 구독에 적합.
- **Per-project / Per-protocol**: 클린리서치 등 프로젝트·스터디당 고정비. 제약 고객의 “이번 프로젝트 예산”과 잘 맞음.
- **Per-outcome / Usage**: 사용량(리포트 건수, API 호출 등) 기반. 예측은 어렵지만 공정한 체감가격.

**시사점**: 아그와헬스는 **Per-project(원타임) + 팀 구독** 조합이 국내 CEO·지사에 가장 설명하기 쉽고, “클릭해서 한 건 신청”이 가능한 구조가 유리.

### 2.3 Good–Better–Best (B2B 티어)
- **중간 티어가 전환율이 가장 높음**(극단 회피, 앵커 효과). “가장 많이 선택하는 플랜”을 **성장형(Standard)**에 두는 것이 유리.
- **가격 비율** 참고: Good : Better : Best ≈ 1 : 1.6 : 2.5.
- **티어는 3~4개**가 적당. 5개 이상이면 선택 피로.

## 3. 국내 CEO 관점에서 “클릭해서 구매하고 싶은” 서비스 모형 제안

### 3.1 원칙
1. **진입 장벽 최소화**: “전체 플랜 상담”만 있지 않고, **지금 당장 한 가지라도 선택 가능**하게 (원타임 단품 + 구독 티어).
2. **가치 선노출**: 각 옵션에 “무엇을 얻는지”(결과물·건수·지원 수준)를 명시해, CEO가 “우리에게 맞는 건가?”를 바로 판단하게.
3. **신뢰·투명**: “구체 금액은 문의 후 제안”이라도, **가격대 힌트**(예: “프로젝트당 000만원대~”) 또는 **비교 메시지**(“글로벌 컨설팅 대비 1/N 수준”)를 주면 결재 심리가 줄어듦.
4. **다음 행동 명확**: 카드마다 **“이 플랜 문의하기”**, **“이 리포트 1건 신청”** 등 한 가지 CTA.

### 3.2 제안 서비스 형태 (웹에 노출)

| 구분 | 내용 | 클릭 시 동작 |
|------|------|----------------|
| **원타임 단품** | 예상 보험가격 1건, 외국현황 리포트 1건, 퇴장방지 원가 1건 등 **모듈별 1건 단위** | 문의 모달 열기 + “관심 서비스: OO 1건” 자동 기입 |
| **스타터(입문)** | 핵심 AI 모듈 일부, 월 N건 또는 프로젝트 1~2건 수준 | “이 플랜 문의하기” → 모달에 플랜명 전달 |
| **그로스(성장)** ⭐ | 의약품·의료기기 대부분 모듈, 팀 구독 + 원타임 추가 | 동일, “가장 많이 선택” 뱃지 |
| **스케일(확장)** | 전 모듈·커스텀·전담 컨설턴트 | “이 플랜 문의하기” |
| **비교표** | 티어별 “포함 모듈·건수·지원” 한눈에 비교 | 표 아래 CTA |

### 3.3 수익 구조 정리
- **1차 수익**: 원타임 단품(건당) + 구독(월/연).
- **2차 수익**: 구독 가입 후 추가 프로젝트·추가 건수 과금.
- **전환 설계**: 무료 데모/체험 → 원타임 1건 결제 → 동일 모듈 반복 시 구독 제안 또는 건수 패키지 제안.

## 4. 웹 UI 반영 체크리스트

- [ ] 가격 섹션 제목: CEO 관점 메시지 (예: “예산에 맞는 옵션을 선택하세요”).
- [ ] 원타임 단품 카드/리스트: 모듈별 “1건 신청” 버튼, 클릭 시 문의 모달 + 서비스 항목 자동 기입.
- [ ] 구독 티어 3개: 스타터 / 그로스(추천) / 스케일, 카드당 “이 플랜 문의하기” 버튼.
- [ ] (선택) 가격대 힌트 또는 “문의 후 48시간 내 제안” 등 신뢰 문구.
