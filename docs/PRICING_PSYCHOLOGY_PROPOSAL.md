# 심리 원칙 기반 Pricing 제안

진실만 사용하는 전제 하에, 심리 효과를 적용한 가격·문구·구성 제안입니다.

---

## 1. 적용할 심리 원칙과 구체 제안

| 원칙 | 적용 위치 | 제안 (진실만) |
|------|-----------|----------------|
| **앵커링 (Anchoring)** | 섹션 상단·가격 인식 | “글로벌 마켓액세스 컨설팅은 프로젝트당 수천만원~억 단위가 일반적”처럼 **사실인** 비교 기준을 먼저 제시 → 우리 가격대가 상대적으로 낮게 느껴지게. (실제 시장 조사 근거 있을 때만 사용) |
| **손실 회피 → 이득 프레이밍** | 티어 설명·가격 문구 | “비용” 대신 “절감·효과”로 표현. 예: “리포트 1건당 00만원대 — 내부 인력 N시간 대체 가능” (실제 가능한 범위만) |
| **Good–Better–Best / 중앙 무대** | 티어 3개, 중간 추천 | 스타터 / **그로스(가장 많이 선택)** / 스케일. 중간 플랜을 시각·뱃지로 강조해 “안전한 선택”으로 인식시키기. |
| **미끼 효과 (Decoy)** | 비교표·티어 구성 | 스케일(고가)을 두어 그로스가 “적당한 절충”으로 보이게. 세 플랜 모두 **실제 제공 가능한** 옵션이어야 함. |
| **호혜 (Reciprocity)** | 원타임·상담 전 | “무료 데모·체험” 또는 “무료 맞춤 견적(48시간 내)”를 **진짜로** 제공 → 부담 없이 받게 한 뒤, 결제는 선택으로. |
| **기본 편향 (Default)** | 문의 플로우 | 문의 폼 기본 선택을 “그로스 플랜 문의”로 두기. (실제로 가장 많이 문의되는 플랜이면 OK) |
| **인식 우선 (Recognition over Recall)** | 결제·문의 직전 | “선택하신 플랜: 그로스 — 월 100만원대~, 온보딩 포함”처럼 **방금 본 내용**을 짧게 다시 노출. |
| **탈출 경로 (Exit Points)** | 상담·문의 문구 | “상담 후 진행 여부는 전적으로 귀사가 결정합니다. 의무 없음.”을 명시해 불안 해소. |
| **피크엔드 규칙 (Peak-End)** | 체험·데모 설계 | 데모의 **끝**을 “다음 단계: 1건 신청 또는 플랜 견적”으로 정리. 강압 없이 한 줄 CTA만. |
| **점진적 노출 (Progressive Disclosure)** | 비교표 | 포함 모듈 → 사용 규모 → 지원 수준 → 가격대 순으로 **한눈에** 비교. 정보 과부하 줄이기. |
| **넛지 (Nudge)** | CTA | “이 플랜 문의하기”를 눈에 띄게 하되, “안 해도 불이익 없음”을 근처에 표기. |
| **프레이밍 (Framing)** | 가격대 문구 | “월 50만원대~” 대신 “월 50만원대부터 (규모에 따라 조정)”처럼 **투명·유연**하게. |
| **제이가르닉 효과** | 원타임 후 구독 | “한 건 체험 후, 같은 모듈을 반복 쓰시면 구독이 더 유리할 수 있습니다” — **사실**이면 안내. |

## 2. aiPricing.json에 넣을 문구 제안 (진실만)

### 2.1 섹션 레벨

- **sectionTitle**: “예산에 맞는 옵션을 선택하세요” (유지)
- **sectionSubtitle**: “구독으로 안정적으로 쓰거나, 필요한 1건만 먼저 써보세요. 글로벌 컨설팅 대비 부담 없는 가격대로 설계했습니다.” (유지 — 앵커 + 프레이밍)
- **anchorLine** (신규): “일반적인 마켓액세스 컨설팅은 프로젝트당 수천만원대입니다. AI로 같은 결과물을 더 낮은 비용으로.”  
  → **실제 시장 조사 근거가 있을 때만** 사용. 없으면 생략.
- **reciprocityLine** (신규): “무료로 견적부터 받아보세요. 48시간 내 제안드립니다.”
- **exitLine** (신규): “상담 후 진행 여부는 귀사가 결정합니다. 별도 의무 없습니다.”

### 2.2 원타임

- **oneTime.subtitle**: “한 건만 필요할 때. 결재도 빠르고, 체험 후 구독으로 전환할 수 있습니다.” (유지 — 호혜·탈출)
- **oneTime.priceHint**: “1건당 견적 문의 (서비스별 상이)” (유지)

### 2.3 티어 (Good–Better–Best)

- **그로스**에만 **badge**: “가장 많이 선택” (실제 데이터 있을 때만)
- **gainFraming** (티어별, 선택):  
  - 스타터: “내부 리소스 한 건 분량을 이 가격대로 대체할 수 있습니다.”  
  - 그로스: “팀 전체가 쓰기에 맞는 가성비.”  
  → **실제로 가능한 범위**만 기재.

### 2.4 비교표

- **comparisonTable.subtitle**: “포함 모듈·건수·지원 수준을 한눈에 비교하세요.” (유지 — 점진적 노출)
- 행 순서: 포함 AI 모듈 → 사용 규모 → 지원 수준 → 가격대 (유지)

### 2.5 신뢰·탈출

- **trustNote**: “구체적인 금액은 제품 포트폴리오와 사용 규모를 확인한 뒤 48시간 내 투명하게 제안드립니다.”
- **exitLine**: “상담 후 진행 여부는 전적으로 귀사가 결정합니다. 별도 의무 없습니다.”

## 3. UI 배치 제안

1. **섹션 최상단**: sectionTitle + sectionSubtitle (현재 유지)
2. **앵커 라인**: subtitle 바로 아래, 작은 글씨로 anchorLine (사실일 때만)
3. **원타임**: 현재 유지 + reciprocityLine을 원타임 그리드 위 또는 아래 한 줄
4. **구독 티어**: 현재 유지 (그로스 추천 뱃지, priceHint)
5. **비교표**: 현재 유지
6. **trustNote** 아래에 **exitLine** 한 줄 추가
7. **CTA**: “플랜 비교·맞춤 상담 요청” 유지, 옆에 작게 “(상담만 받고 결정하지 않아도 됩니다)” 가능

---

## 4. 사용하지 않을 것 (거짓·과장 방지)

- 거짓 결핍 (“남은 N자리” — 실제 한정이 아니면 사용 금지)
- 거짓 사회적 증거 (실제 고객 수·사례가 없으면 “많은 팀이 선택” 등 사용 금지)
- 거짓 긴급성 (“오늘만” — 실제 마감이 아니면 사용 금지)
- 반발 유발 (“안 하면 손해” 등 강압 문구 금지)

---

## 5. 체크리스트 (배포 전)

- [ ] anchorLine 사용 시: 시장 조사·근거 있음
- [ ] “가장 많이 선택”: 실제 문의/가입 데이터로 확인
- [ ] gainFraming 문구: 내부 검토로 “가능한 범위”만 기재
- [ ] exitLine: 모든 상담·문의 영역 근처에 노출
- [ ] 가격대(월 50만원대~ 등): 실제 제공 가능 구간과 일치
