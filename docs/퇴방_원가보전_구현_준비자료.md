# 퇴방(퇴장방지의약품) 원가보전 웹 구현 – 준비해주시면 좋은 자료

웹에서 **서식·자료입력·원가계산**까지 구현하려면 아래를 준비해 주시면 설계와 개발에 반영할 수 있습니다.

---

## 1. 서식·가이드 (필드 정의용)

| 준비해 주실 것 | 용도 |
|----------------|------|
| **HIRA/복지부 퇴장방지의약품 관련 서식** (PDF·엑셀 등) | 웹 입력 항목을 공식 서식과 1:1로 맞추기 위함 |
| **원가산정 시 제출하는 서류 목록** (체크리스트) | “필수 입력 항목” vs “선택/첨부” 구분 |
| **회사 내부 사용 템플릿** (엑셀 등) | 실제 업무에서 쓰는 항목명·순서·단위 반영 |

**없다면:** 공개된 HIRA “산정대상약제”·“원가산정” 관련 안내 페이지 URL만 있어도, 그 기준으로 필드 목록을 추려서 제안할 수 있습니다.

---

## 2. 원가계산 로직 (산출 공식·규칙)

웹에서 “원가계산” 버튼을 눌렀을 때 **어떤 식으로 금액을 만드는지**가 필요합니다.

| 준비해 주실 것 | 예시 |
|----------------|------|
| **산정 단가 산출 공식** | 예: 제조원가 + 유통단계 비용 + … = 1단위당 산정금액 |
| **단계별 항목** | 제조원가, 포장단위, 유통마진, 기타 반영 항목 등 |
| **반올림·단위 규칙** | 예: 원 단위, 10원 단위, 1캡슐/1앰플 등 |
| **계산 예시 1건** | “이렇게 넣으면 이렇게 나온다” 한 건 (숫자만 가명으로) |

**형식:**  
- 엑셀에 공식이 있으면 그 시트(또는 수식 설명)  
- 문서(한글/워드)에 적힌 산출 방법  
- “이 약은 이렇게 계산한다”는 메모  

**가능하면:**  
- **입력값 → 중간 단계 → 최종 산정금액**이 한 줄로 보이는 예시 1건  
- “이 경우엔 A 공식, 이 경우엔 B 공식” 같은 **분기 조건**이 있다면 간단 설명  

이걸 기준으로 웹에서 동일한 계산이 나오도록 구현합니다.

---

## 3. 입력 항목 정의 (데이터 구조)

아래는 “일반적으로 퇴방 원가보전에 필요할 수 있는 항목” 예시입니다.  
**실제로 쓰는 항목·명칭·필수 여부**를 알려주시면, 그에 맞춰 서식과 입력 화면을 설계합니다.

### 3.1 제품·품목 정보
- 제품명(또는 가칭)
- 성분명
- 포장 단위 (1캡슐, 1앰플, 1병 등)
- 1회 투여량·1일 투여 횟수 (필요한 경우)
- 기타 (표준코드, 품목허가 관련 등)

### 3.2 원가 관련
- **제조원가** (단위당 또는 포장당)
- **유통 단계** (도매·약국 등)별 비용 또는 마진
- **기타 반영 비용** (있을 경우: 운송, 보관, 인건비 등)
- **산정 요청 단위** (1캡슐 기준인지, 1포장 기준인지 등)

### 3.3 제출·관리용
- 지정 희망 시점
- 담당자·메모
- 첨부 파일(원가명세서, 증빙 등)

**요청:**  
- 위 항목 중 **실제로 사용하는 것만** 골라 주시거나,  
- “우리 쪽에는 여기에 있는 것 말고 ○○가 더 필요하다”라고 알려 주시면 됩니다.

---

## 4. 예시 데이터 (1~2건)

| 준비해 주실 것 | 용도 |
|----------------|------|
| **가명 예시 1건** | 입력 화면·서식 배치 검증, 원가계산 결과 비교 |
| **“정답” 산정 결과** | 같은 입력에 대해 회사/엑셀에서 나온 금액 (웹 계산 검증용) |

개인정보·실제 약가가 들어가지 않도록 **완전 가명·가상 숫자**로만 주셔도 됩니다.

---

## 5. 정리하면

| 순서 | 준비 자료 | 있으면 좋은 정도 |
|------|-----------|------------------|
| 1 | **원가계산 공식/규칙** (또는 엑셀 수식·설명) | ★ 필수 – 이게 있어야 “원가계산” 구현 가능 |
| 2 | **실제 사용하는 입력 항목 목록** (또는 공식 서식) | ★ 필수 – 서식·입력 필드 설계 기준 |
| 3 | **계산 예시 1건** (입력값 + 기대 결과) | ★ 권장 – 구현 검증용 |
| 4 | HIRA/복지부 서식·가이드 PDF·URL | 있으면 서식과 완전히 맞출 수 있음 |
| 5 | 회사 내부 템플릿(엑셀 등) | 있으면 업무 흐름에 맞춘 UI 설계 가능 |

---

## 6. 준비해 주신 뒤 진행 방식

1. **공식/규칙 + 항목(또는 서식)** 주시면 →  
   “입력 화면 설계안 + 원가계산 로직 설계안”을 제안드립니다.

2. **예시 1건(입력 + 정답)** 주시면 →  
   웹 계산 결과가 예시와 일치하는지 검증할 수 있습니다.

3. 이후 단계:  
   - 웹 **서식(단계별 입력 화면)** 구현  
   - **원가계산** 버튼 → 동일 공식으로 산출  
   - 결과 **다운로드**(엑셀/PDF 등)는 요구사항에 맞춰 추가

위 항목 중에서 **지금 갖고 계신 것**만 알려 주셔도 됩니다. 그걸 기준으로 다음 단계(서식 설계 또는 계산 로직 구현)부터 구체적으로 잡을 수 있습니다.
